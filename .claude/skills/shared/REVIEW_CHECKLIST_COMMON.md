## 1. セキュリティ

### Must（必須）

- [ ] **認証情報がハードコードされていない**
  - `.env` は `.gitignore` 済み

- [ ] **SQLインジェクション対策: パラメータ化クエリを使用している**
  - 違反例: 文字列結合で SQL を構築
  - 正解: プレースホルダー（`$1`, `?` 等）を使用

- [ ] **認可チェック（IDOR防止）: 自分のリソースのみアクセス可能**
  - リソース取得後に所有者チェック、または WHERE 条件に UserID を含める

- [ ] **パスワードが平文で保存・ログ出力されていない**
  - bcrypt 等でハッシュ化必須

- [ ] **JWT / トークンの安全な取り扱い**
  - 署名アルゴリズム検証
  - `none` 署名を受け入れない
  - トークン種別（access/refresh）の混同防止

- [ ] **入力検証がある（APIエンドポイント、外部入力）**
  - リクエストボディサイズ制限

- [ ] **CORS 設定でワイルドカード（`*`）を使用していない**
  - 許可オリジンを明示的に指定

- [ ] **コマンドインジェクション対策**
  - ユーザー入力をシェルコマンドに直接渡さない

### Should（推奨）

- [ ] **エラーメッセージが内部情報を露出していない**
  - スタックトレース、DB 構造、内部 IP 等

- [ ] **機密情報がログに含まれていない**
  - パスワード、トークン、個人情報

- [ ] **既知の脆弱性を持つ依存ライブラリを使用していない**

## 2. パフォーマンス

### Must（必須）

- [ ] **N+1 問題がない**
  - ループ内で DB/API 呼び出しがない
  - 違反例: `for item in items: db.get(item.id)`
  - 正解: バッチ取得または JOIN

- [ ] **1リクエストあたりのDBクエリ数が適切（目安: 5以下）**

### Should（推奨）

- [ ] **不要な SELECT * / 全カラム取得を使用していない**
  - 必要なカラムのみ取得

- [ ] **外部呼び出しに適切なタイムアウトが設定されている**

- [ ] **トランザクション範囲が最小限である**
  - 長時間ロック防止

- [ ] **コネクション / リソースリークがない**
  - `defer Close()` / `try-finally` / `with` の適切な使用

- [ ] **大量データの一括読み込みを回避している**
  - ページネーションやストリーム処理

### Nice（改善）

- [ ] **静的データをキャッシュしている**
  - マスタデータ等の変更頻度が低いデータ

## 3. ドキュメント

### Should（推奨）

- [ ] **破壊的変更時にドキュメントが更新されている**
- [ ] **新しい環境変数を追加した場合に設定例が更新されている**

### Nice（改善）

- [ ] **複雑なロジックにコメントがある（WHY を説明）**

## 4. Git / コミット

### Should（推奨）

- [ ] **コミットメッセージが明確**（Conventional Commits 等）
- [ ] **無関係な変更が含まれていない**

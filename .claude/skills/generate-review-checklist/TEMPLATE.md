# {プロジェクト名} コードレビューチェックリスト

このドキュメントは、{プロジェクト名} プロジェクト固有のレビュー観点をまとめた**共通チェックリスト**です。
`code-review` と `review-pr` の両スキルからこのファイルを参照します。

> **更新方法**: プロジェクトの規約やアーキテクチャが変わった場合は `/generate-review-checklist` を実行して再生成してください。

<!--
  以下のカテゴリはプロジェクトの特性に応じて取捨選択・追加してください。
  各カテゴリ内の項目は Must / Should / Nice の3段階で分類します。

  分類基準:
    Must   — 違反するとバグ・セキュリティ問題・アーキテクチャ崩壊を引き起こす
    Should — 品質・保守性に影響するが致命的ではない
    Nice   — さらに改善できるが対応は任意

  各項目のフォーマット:
    - [ ] **チェック内容**
      - 違反例: `具体的なコード`
      - 正解: `正しいコード`
      - 根拠: どのドキュメント・規約に基づくか
-->
